Create Your GitHub Pages Website

Turn your GitHub repository into a live website in just a few steps

What is GitHub Pages?

GitHub Pages is a free static site hosting service provided by GitHub. It takes HTML, CSS, and JavaScript files from a repository on GitHub, optionally runs the files through a build process, and publishes a website.

You can create sites for yourself or for projects directly from your GitHub repositories.

Step-by-Step Guide

1

Create a GitHub Account

If you don't have one already, sign up for a free account at github.com.

2

Create a New Repository

Create a new repository named username.github.io where "username" is your GitHub username.

Important

This repository name must match exactly, including capitalization.

3

Add Your Website Content

Clone the repository to your computer, add your HTML, CSS, and JavaScript files, then push them back to GitHub.

Your main page should be named index.html.

4

Enable GitHub Pages

In your repository, go to Settings → Pages. Select the main branch as source and click Save.

5

Wait for Deployment

It might take a few minutes for your site to be published. Refresh the page to see when it's live.

6

Visit Your Website

Open a new browser tab and go to https://username.github.io to see your live website!

Example Repository

Your repository structure should look similar to this:

username / username.github.io

├── index.html
├── styles.css
├── script.js
└── assets/
    ├── images/
    └── fonts/

Pro Tip

You can use Jekyll, a static site generator, with GitHub Pages for more advanced functionality. GitHub will automatically build your site when you push changes.

Custom Domain

You can use a custom domain with your GitHub Pages site:

  1. Add a CNAME file with your domain name to your repository
  2. Configure your DNS settings to point to GitHub's servers
  3. Enable the custom domain in your repository settings
